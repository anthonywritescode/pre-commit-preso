Stop wasting time in code reviews

An Introduction to pre-commit

All too often in code reviews

  • Your indentation is off...
  • This doesn't conform to pep8...
  • Your imports are out of order...
  • blah blah blah blah blah

Stop

Tools

  • Linters
    • flake8, pylint, jshint, scss-lint, ...
  • Fixers
    • autopep8, trailing whitespace, end-of-files, encoding markers, import reordering,...

Solution!

git hooks!

Problems

  • Large hardcoded bash script
  • System dependencies
  • Locked to one language
  • Consistency

pre-commit solves this!

$ pip install pre-commit
Check in a .pre-commit-config.yaml 
-   repo: git@github.com:pre-commit/pre-commit-hooks
    sha: v0.4.1
    hooks:
    -   id: trailing-whitespace
    -   id: autopep8-wrapper
    -   id: flake8
-   repo: git@github.com:asottile/reorder_python_imports
    sha: ea9fa14a757bb210d849de5af8f8ba2c9744027a
    hooks:
    -   id: reorder-python-imports
-   repo: git://github.com/pre-commit/mirrors-jshint
    sha: e86225836aadd058036f47a6de9b0ee0440476eb
    hooks:
    -   id: jshint
$ pre-commit install

Features

  • Python (2+3), Ruby, Node Supported
  • System binaries also still work
  • Merge conflicts only check conflicted files
  • Easy to add new linters
  • And more!
